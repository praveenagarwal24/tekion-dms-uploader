workflow
How it works
Seven steps — only one is manual. Everything else runs in a visible Chrome browser so you can watch it work.
01
Scan & unzip
Scans the folder you specify for .zip files, extracts each one into a named subfolder, deletes the originals.
02
Login to Tekion DMS
Opens a real Chrome window, navigates to the DMS URL, fills in your credentials automatically.
03
Enter OTP
Automation pauses and waits. You enter the OTP in the browser, then click "OTP entered" in the UI to resume.
⚡ manual step
04
Navigate to inventory
Clicks the 9-dot app launcher and opens Vehicle Inventory. Falls back to a manual prompt if navigation fails.
05
Search & delete media
For each VIN: searches it, opens the record, clicks the Media tab, selects all images, deletes them via the popup.
06
Upload new images
Clicks Upload Media, opens your local VIN folder, selects all images, and waits for the upload to complete.
07
Save & confirm
Clicks Save, dismisses the upload confirmation popup, repeats until the page returns to the search screen.
features
What's included
Real-time UI
Per-VIN status grid, live activity log, progress bar, and step-by-step nav — all in your browser.
OTP pause & resume
Automation pauses cleanly at the OTP screen and resumes the moment you confirm — no timeouts.
Save retry loop
Handles Tekion's upload confirmation popups automatically — clicks Save until the page navigates away.
Cross-platform
One codebase, two launchers — start_windows.bat and start_mac.sh. Python handles the rest.
Config file
Edit config.json to change credentials, port, or file extensions — no Python knowledge needed.
Stop & reset
Stop mid-run at any time. Failed VINs are highlighted in red so you know exactly what to retry.
